# SuperApp

**SuperApp** is a framework on top of Django, designed to accelerate the development of software projects for startups. It incorporates all the powerful features of Django, plus additional tools to seamlessly integrate various applications.

<Image src="/assets/docs/admin-portal/admin-portal.png" alt="Admin Portal Screenshot" width={1024} height={1024} className={'mt-3'} />


## Get Started

<Steps>
    ### Install

    To create a SuperApp project you need `django_superapp` installed. You can do this
    using `pipx`:

    ```shell
    pipx install django_superapp --force
    ```

    ### Generate the Project

    Next, create a new SuperApp project using the `bootstrap-project` command

    ```shell
    django_superapp bootstrap-project \
        --template-repo https://github.com/django-superapp/django-superapp-default-project \
        ./my_superapp;
    cd my_superapp;
    ```

    With the above command, a new project will be created using `default` template in the `my_superapp` directory.

    ### Add Admin Portal App
    Next, add the `admin_portal` app to the project using the `bootstrap-app` command

    ```shell
    cd superapp/apps;
    django_superapp bootstrap-app \
        --template-repo https://github.com/django-superapp/django-superapp-admin-portal ./admin_portal;
    cd ../../;
    ```

    ### Add Authentication App
    Next, add the `authentication` app to the project using the `bootstrap-app` command

    ```shell
    cd superapp/apps;
    django_superapp bootstrap-app \
        --template-repo https://github.com/django-superapp/django-superapp-authentication ./authentication;
    cd ../../;
    ```

### Start the server
    To get started, first make sure you have [Docker](https://www.docker.com/) and [docker-compose](https://docs.docker.com/compose/) installed on your system.

    Finally, you can start the development server using the following command:

    ```shell
    make setup-sample-env
    make start-docker
    ```

    The above command will start the development server and you can access the project at [http://localhost:8080](http://localhost:8080).


### Apply migrations

    To apply migrations, run the following command:

    ```shell
    docker-compose exec web python3 manage.py migrate;
    ```

### Create a superuser

    To create a superuser, run the following command:

    ```shell
    docker-compose exec web python3 manage.py createsuperuser
    ```
</Steps>
